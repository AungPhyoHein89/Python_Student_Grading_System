# Python Student Grading & Report Card System

A clean, modular, and dynamic student grading system written in Python. This repository documents my progression from basic programming logic to professional-grade implementations using advanced functions, conditional structures, and nested object profiles.

## 🚀 Evolution of the Project

The project is structured across four specific script versions, each demonstrating a unique level of enhancement and code optimization:

### 1. `v1_student_report_card.py`
* **Concept:** Introduction to Nested Object Management.
* **Features:** Utilizes dictionary structures containing nested datasets (student meta-data paired with specific subject marks) to map individual identities with automated grade calculations across dynamic parameters.

### 2. `v2_student_report_card.py`
* **Concept:** Separation of Evaluation Control and Dynamic Variable Mapping.
* **Features:** Introduces complex operational logic to cross-check varying subject thresholds dynamically (e.g., Distinction criteria at 80 for specific languages and 85 for sciences), returning a consolidated cumulative academic final status.

### 3. `v3_clean_grading.py`
* **Concept:** Inline Logical Evaluations.
* **Features:** Focuses on standard input processing sequences, validating individual grade profiles without structural modularity overhead, serving as a clean baseline for functional programming transitions.

### 4. `v4_clean_grading.py`
* **Concept:** Modular Refactoring and Micro-optimizations.
* **Features:** Implements highly reusable standalone function components using immediate validation returns (`return` statement mappings) and precision float formatting (`:.2f`) to handle statistical averages with high precision.

---

## 🛠️ Concepts Demonstrated

* **Data Structures:** Comprehensive handling of Dictionary Iterations (`.items()`), List Membership validation (`in`), and Multi-layered Data Nesting.
* **Algorithmic Logic:** Advanced Conditional Control Blocks (`if-elif-else`), Boolean State Flags (`all_passed`), and Dynamic Parameter Passing.
* **Mathematical Operations:** Automated aggregate totals and precision floating-point arithmetic.
* **Clean Code Architecture:** Alignment with PEP 8 layout conventions, explicit component variable naming, and efficient code paths.
